# 프로젝트 개요 문서 (Project Overview)

## 1. 프로젝트 기본 정보
* **프로젝트명:** PSM (Project Script Manager) *[가칭]*
* **한 줄 소개:** 비주얼 노벨 게임 개발자를 위한 시나리오 그래프 및 리소스 통합 관리 웹 서비스
* **목표 플랫폼:** 웹 서비스 (Web Service)
* **릴리즈 버전:** v1.0.0 (초기 버전)

## 2. 기획 배경 및 목적
* **배경:** 비주얼 노벨 및 스토리 중심 게임 개발 시, 복잡해지는 인물 간의 대화, 캐릭터의 행동 변화, 유기적인 선택지, 그리고 이에 배경·음향·일러스트 등의 리소스를 매칭하고 관리하는 작업의 복잡도가 매우 높음.
* **목적:** * 스토리 흐름(시나리오)의 시작과 끝을 시각적인 그래프(Graph) 형태로 설계하여 직관적인 개발 환경 제공.
  * 인물, 세계관 설정, 상호작용 및 게임 리소스를 한곳에서 쉽게 생성하고 체계적으로 관리하여 개발 효율성 극대화.

## 3. 타겟 사용자 (Target Audience)
* **v1.0.0 핵심 타겟:** 1인 게임 개발자
  * 복잡한 협업 기능보다는 직관적인 UI와 빠른 시나리오 스케치, 데이터 관리의 편의성을 중시하는 개인 개발자를 핵심 대상으로 설정.

## 4. 핵심 기능 (Core Features) - v1.0.0
* **시나리오 그래프 에디터:** 노드(Node) 기반으로 시나리오의 흐름, 선택지, 분기점을 시각적으로 편집하고 직관적으로 흐름을 파악하는 기능.
* **리소스 및 데이터 관리:** 인물 설정, 세계관(로어), 이미지/사운드 리소스를 스크립트 노드와 쉽게 연동하고 관리하는 기능.
* **데이터 내보내기 (Export):** 작성한 시나리오 및 리소스 데이터를 게임 엔진이나 다른 도구에서 활용할 수 있도록 **JSON** 및 **CSV** 형식으로 추출하는 기능.

## 5. 기술 스택 (Tech Stack)
* **Frontend:** React
* **Backend:** Express (Node.js)
* **Database:** PostgreSQL

## 6. 모듈 구성 및 관계 (Module Architecture)
PSM 서비스의 핵심 모듈은 전체 게임을 총괄하는 `Project`를 최상위 단위로 하며, 데이터의 재사용성과 유기적인 연동을 위해 다음과 같은 구조적 관계를 가집니다.

### 6.1 모듈 정의
* **Project (프로젝트):** 사용자가 생성하는 개별 게임 개발 작업의 최상위 독립 단위 모듈입니다.
* **Story (스토리):** 한 프로젝트 내의 독립적인 시나리오 챕터 또는 에피소드 흐름을 관리하는 모듈입니다.
* **Resource (리소스):** 게임에 공통으로 사용되는 배경 이미지, BGM, 효과음 등 미디어 자원을 관리합니다.
* **Character (캐릭터):** 인물 정보, 이름, 대사 속성 등 인물의 기본 설정을 관리합니다.
* **Character Pose (캐릭터 포즈):** 특정 캐릭터에 종속된 표정, 의상, 자세 등 세부 스탠딩 일러스트를 관리합니다.

### 6.2 모듈 관계 다이어그램 (Module Relations)
```
         [ Project ] (최상위 독립 단위)
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 [Story]  [Resource] [Character]
    │         ▲         │
    │         │         ▼
    │         │   [Character Pose]
    │         │         ▲
    └─────────┼─────────┘ (참조 및 연출 결합)
              ▼
        [Graph_Node]
```

### 6.3 데이터 연동 및 참조 규칙
1. **독립적 관리, 유기적 참조:** `Resource`와 `Character`는 프로젝트 전체에서 재사용되므로 `Project`에 직속 저장됩니다. 대신, 스토리의 실제 연출을 담당하는 `Graph_Node`에서 이들을 필요한 시점에 **'참조(Reference)'**합니다.
2. **캐릭터 포즈의 수직 종속:** `Character Pose`는 `Character` 모듈에 강하게 종속됩니다. 특정 캐릭터가 삭제되면 해당 캐릭터의 포즈 데이터도 함께 정리되어야 합니다.

## 7. 데이터베이스 설계 방향 (Database Schema)
* **DBMS:** PostgreSQL
* **설계 특징:** 관계형 데이터베이스(RDB)의 외래키(FK) 제약조건을 활용하여 데이터 무결성을 보장하며, 그래프 에디터의 유연한 데이터 표현(좌표, 커스텀 속성 등)을 위해 `JSONB` 타입을 적극 활용합니다.

### 7.1 엔티티 관계도 (Conceptual ERD)
```
 [ Project ]
      │
      ├───(1:N)───► [ Story ] ───(1:N)───► [ Graph_Node ] ◄───(N:M)─── [ Graph_Edge ]
      │                                         │
      │ (소유)                                  │ (참조)
      ├───(1:N)───► [ Resource ] ◄──────────────┤
      │                                         │ (참조)
      └───(1:N)───► [ Character ] ───(1:N)───► [ Character_Pose ]
```

### 7.2 테이블 정의 요약
1. **projects (프로젝트):** 사용자가 생성한 비주얼 노벨 프로젝트의 최상위 마스터 테이블 (ID, 프로젝트명, 생성일 등)
2. **stories (스토리):** 각 프로젝트 내 속한 시나리오 대단위 챕터/에피소드 정보
3. **graph_nodes (그래프 노드):** 시각적 화면 좌표(X, Y), 노드 타입(시작, 대사, 선택지, 연출, 끝)을 가지며, 특정 `Character`, `Character_Pose`, `Resource`의 ID를 외래키로 참조하여 대사와 연출을 결합
4. **graph_edges (그래프 엣지):** 소스 노드와 타겟 노드 간의 연결선 정보 및 선택지 클릭 시 이동할 조건 분기 데이터 관리
5. **resources (리소스):** 프로젝트에 등록된 이미지/사운드 파일의 메타데이터 및 클라우드 저장소 경로(URL) 관리
6. **characters (캐릭터):** 프로젝트에 속한 등장인물의 이름, 대사 창 표시명, 대표 색상 등 관리
7. **character_poses (캐릭터 포즈):** 특정 캐릭터에 종속된 일러스트 정보로, 이미지 리소스(`resources.id`)와 1:1 또는 1:N 연동
